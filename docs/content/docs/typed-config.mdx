**Typed Config** provides schema-driven configuration loading with strict type validation, enum constraints, range checks, and sensitive field tracking. Available in all 8 languages since v0.1.2.

## Overview

Instead of loading raw config and casting manually, define a schema and let DOTLYTE validate and coerce everything for you:

<Tabs items={["Python", "JavaScript", "Go", "Rust", "Java", "Ruby", "PHP", "C#"]}>
  <Tab value="Python">
    ```python
    from dotlyte import TypedConfig

    config = TypedConfig.create({
        "PORT": {"type": "integer", "required": True, "min": 1, "max": 65535},
        "DEBUG": {"type": "boolean", "default": False},
        "LOG_LEVEL": {"type": "string", "enum": ["debug", "info", "warn", "error"], "default": "info"},
        "API_KEY": {"type": "string", "required": True, "sensitive": True},
    })

    config["PORT"]      # 8080 (int, validated)
    config["DEBUG"]     # False (bool, from default)
    config["LOG_LEVEL"] # "info" (string, enum-checked)
    ```

## Field Descriptor

Each key in the schema maps to a **FieldDescriptor** with these properties:

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `type` | `string` | `"string"` | One of: `"string"`, `"integer"`, `"float"`, `"boolean"`, `"array"` |
| `required` | `boolean` | `false` | Throw if the key is missing from env |
| `default` | `any` | — | Fallback value when env var is not set |
| `enum` | `string[]` | — | Allowed values (post-coercion for strings) |
| `min` | `number` | — | Minimum value (for `integer` and `float`) |
| `max` | `number` | — | Maximum value (for `integer` and `float`) |
| `sensitive` | `boolean` | `false` | Marks the field as containing a secret |
| `doc` | `string` | — | Human-readable documentation for this field |

## Type Coercion

Typed Config reads from environment variables (strings) and coerces them:

| Type | Input | Output |
|------|-------|--------|
| `integer` | `"8080"` | `8080` |
| `float` | `"3.14"` | `3.14` |
| `boolean` | `"true"`, `"yes"`, `"1"`, `"on"` | `true` |
| `boolean` | `"false"`, `"no"`, `"0"`, `"off"` | `false` |
| `array` | `"a,b,c"` | `["a", "b", "c"]` |
| `string` | `"hello"` | `"hello"` |

## Validation

Typed Config validates after coercion:

- **Required check**: throws if the key is missing and has no default
- **Enum check**: throws if the coerced value is not in the allowed list
- **Min/Max check**: throws if the numeric value is out of range
- **Type check**: throws if the raw value cannot be coerced to the target type

All validation errors are collected and thrown as a single `DotlyteError` with all messages.

### Skip Validation

Pass `skip_validation: true` to bypass validation (useful for testing):

```python
config = TypedConfig.create(schema, skip_validation=True)
```

## Sensitive Field Tracking

Fields marked `sensitive: true` integrate with the audit callback:

```python
accessed = []
config = TypedConfig.create(
    {"API_KEY": {"type": "string", "sensitive": True}},
    on_secret_access=lambda key: accessed.append(key),
)

config["API_KEY"]  # Triggers the callback
print(accessed)    # ["API_KEY"]
```

This is useful for audit logging — know exactly when and where secrets are read.

## Error Messages

Typed Config produces clear, actionable error messages:

```
Required config key 'DATABASE_URL' is missing. Set it as an environment variable.
```

```
Config key 'LOG_LEVEL': value 'verbose' is not in allowed values: debug, info, warn, error
```

```
Config key 'PORT': value 0 is less than minimum 1
```

```
Config key 'PORT': cannot convert 'abc' to integer
```
