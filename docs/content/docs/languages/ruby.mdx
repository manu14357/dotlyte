## Installation

```bash
gem install dotlyte
```

Or in your Gemfile:

```ruby
gem "dotlyte", "~> 0.1"
```

Then:

```bash
bundle install
```

### Requirements

- Ruby **3.0+**
- Optional: `yaml` (stdlib), `json` (stdlib)

## Quick Start

```ruby
require "dotlyte"

config = Dotlyte.load

puts config.database.host    # "localhost"
puts config.database.port    # 5432
```

## API Reference

### `Dotlyte.load(**options)`

All parameters are **keyword arguments**:

```ruby
# Basic
config = Dotlyte.load

# With options
config = Dotlyte.load(
  files: [".env", "config.yaml"],
  prefix: "APP_",
  defaults: { port: 3000 },
  env: "production"
)
```

**Returns:** `Dotlyte::Config` instance.

#### Load Options

| Option | Type | Default | Description |
|---|---|---|---|
| `files` | `Array<String>` | `nil` | Explicit config file paths |
| `prefix` | `String` | `nil` | Env var prefix to strip |
| `defaults` | `Hash` | `nil` | Default values (lowest priority) |
| `sources` | `Array<String>` | `nil` | Custom source load order |
| `env` | `String` | `nil` | Environment name |
| `schema` | `Hash` | `nil` | Schema rules for validation |
| `strict` | `Boolean` | `false` | Reject unknown keys |
| `interpolate_vars` | `Boolean` | `true` | Enable `${VAR}` interpolation |
| `overrides` | `Hash` | `nil` | Highest priority values |
| `debug` | `Boolean` | `false` | Debug logging |
| `find_up` | `Boolean` | `false` | Walk up directories |
| `root_markers` | `Array<String>` | `nil` | Project root markers |
| `cwd` | `String` | `nil` | Working directory |
| `allow_all_env_vars` | `Boolean` | `false` | Disable env var filtering |
| `watch` | `Boolean` | `false` | Enable file watching |
| `debounce_ms` | `Integer` | `100` | Watcher debounce (ms) |
| `custom_sources` | `Array` | `nil` | Custom source plugins |

### `Config`

#### `get(key, default = nil)`

```ruby
host = config.get("server.host", "0.0.0.0")
port = config.get("server.port", 8080)
missing = config.get("nope")    # nil
```

#### `require(key)`

Returns the value or raises `Dotlyte::MissingKeyError`:

```ruby
begin
  db_url = config.require("database_url")
rescue Dotlyte::MissingKeyError => e
  puts "Missing: #{e.key}"
  puts "Checked: #{e.sources_checked}"
  exit 1
end
```

#### `require_keys(*keys)`

Batch-require multiple keys. Returns an `Array` of values:

```ruby
host, port, name = config.require_keys("db.host", "db.port", "db.name")
```

#### `has?(key)`

```ruby
if config.has?("redis.host")
  setup_redis(config)
end
```

#### `scope(prefix)`

Create a scoped sub-config:

```ruby
db = config.scope("database")
db.get("host")  # same as config.get("database.host")
db.host          # dot-notation works too
```

#### Dot-Notation (method_missing)

```ruby
# These are equivalent
config.database.host
config.get("database.host")
```

Nested hashes return a new `Config`, allowing chained access.

#### Serialization & Export

```ruby
config.keys              # Array — top-level keys
config.to_flat_keys      # Array<String> — all flattened keys
config.to_flat_hash      # Hash — flat key→value
config.to_h              # Hash — deep nested copy
config.to_h_redacted     # Hash — sensitive values masked
config.to_json           # String — JSON
config.write_to("out.json")  # Write to file (JSON or YAML based on extension)
```

#### Schema Validation

```ruby
schema = {
  "port" => Dotlyte::SchemaRule.new(type: "integer", required: true, min: 1, max: 65535),
  "host" => Dotlyte::SchemaRule.new(type: "string", required: true)
}

violations = config.validate(schema)
violations.each { |v| puts "#{v.key}: #{v.message}" }

config.assert_valid!(schema)  # raises Dotlyte::ValidationError
```

#### File Watching (Hot-Reload)

When `watch: true`:

```ruby
config = Dotlyte.load(watch: true, debounce_ms: 200)

# ConfigWatcher runs on a background thread
# Config refreshes automatically when files change

# When done:
config.close
```

### Variable Interpolation

```bash
# In .env:
BASE_URL=https://api.example.com
API_ENDPOINT=${BASE_URL}/v1
GREETING=${NAME:-World}
REQUIRED=${SECRET:?must be set}
LITERAL=$$NOT_INTERPOLATED
```

### Encryption (AES-256-GCM)

```ruby
require "dotlyte"

key = Dotlyte::Encryption.generate_key

encrypted = Dotlyte::Encryption.encrypt_value("s3cret", key)  # "ENC[...]"
plain = Dotlyte::Encryption.decrypt_value(encrypted, key)       # "s3cret"

Dotlyte::Encryption.encrypt_file(".env", ".env.enc", key)
Dotlyte::Encryption.decrypt_file(".env.enc", ".env.dec", key)
```

Values wrapped in `ENC[...]` are auto-decrypted during `load`.

### Sensitive Value Masking

```ruby
config.to_h_redacted
# {"db.host" => "localhost", "db.password" => "[REDACTED]"}
```

Sensitive keys are auto-detected by patterns: `password`, `secret`, `token`, `api_key`, etc.

### Error Types

All extend `Dotlyte::Error` (which extends `StandardError`):

| Error Class | Extra Properties |
|---|---|
| `Dotlyte::Error` | `key` |
| `Dotlyte::MissingKeyError` | `sources_checked` |
| `Dotlyte::ParseError` | — |
| `Dotlyte::FileError` | `file_path` |
| `Dotlyte::ValidationError` | `violations` |
| `Dotlyte::InterpolationError` | — |
| `Dotlyte::DecryptionError` | — |

```ruby
begin
  config.require("missing")
rescue Dotlyte::MissingKeyError => e
  puts e.key              # "missing"
  puts e.sources_checked  # [".env", "config.yaml"]
rescue Dotlyte::Error => e
  puts e.message
end
```

## Patterns

### Rails Initializer

```ruby
# config/initializers/dotlyte.rb
require "dotlyte"

DOTLYTE_CONFIG = Dotlyte.load(
  prefix: "RAILS_APP_",
  env: Rails.env
)

Rails.application.config.x.dotlyte = DOTLYTE_CONFIG
```

```ruby
# Anywhere in your Rails app
Rails.configuration.x.dotlyte.get("database.host")
```

### Sinatra

```ruby
require "sinatra"
require "dotlyte"

config = Dotlyte.load(prefix: "APP_")

set :port, config.get("port", 4567)
set :bind, config.get("host", "0.0.0.0")

get "/" do
  "Hello from DOTLYTE!"
end
```

### Rake Tasks

```ruby
require "dotlyte"

namespace :db do
  task :migrate do
    config = Dotlyte.load(env: ENV["RACK_ENV"] || "development")
    db_url = config.require("database_url")
    sh "DATABASE_URL=#{db_url} rake db:migrate"
  end
end
```

### RSpec Testing

```ruby
RSpec.describe "Config loading" do
  let(:config) do
    Dotlyte.load(
      defaults: { "database.host" => "localhost", "database.port" => 5432 }
    )
  end

  it "returns default host" do
    expect(config.get("database.host")).to eq("localhost")
  end

  it "raises on missing required key" do
    expect { config.require("missing") }.to raise_error(Dotlyte::MissingKeyError)
  end
end
```

## File Structure

```

├── dotlyte.rb              # Main module and load()
└── dotlyte/
    ├── config.rb           # Config class
    ├── coercion.rb         # Type coercion engine
    ├── errors.rb           # Error hierarchy
    ├── loader.rb           # File loading logic
    ├── merger.rb           # Deep merge
    ├── interpolation.rb    # ${VAR} interpolation
    ├── encryption.rb       # AES-256-GCM encrypt/decrypt
    ├── validator.rb        # Schema validation
    ├── masking.rb          # Sensitive value redaction
    ├── watcher.rb          # File watching / hot-reload
    └── parsers/            # File format parsers
```
