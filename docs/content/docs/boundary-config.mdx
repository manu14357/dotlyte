title: Boundary Config

**Boundary Config** splits your configuration into `server` and `client` contexts, preventing accidental exposure of secrets to client-side code. Available in all 8 languages since v0.1.2.

## The Problem

In full-stack apps, it's easy to accidentally bundle server secrets into client JavaScript:

```javascript
// DANGEROUS — DATABASE_URL gets bundled into the browser!
const config = load();
console.log(config.get("DATABASE_URL"));
```

Boundary Config solves this by enforcing hard boundaries at the config layer.

## How It Works

1. **Define boundaries**: Specify which keys belong to the server and which to the client.
2. **Load config**: Boundary Config loads all config values normally via `load()`.
3. **Gate access**: `server()` returns a view allowing only server keys. `client()` returns a view allowing only client keys.
4. **Hard fail**: Accessing a key from the wrong boundary throws a `DotlyteError` immediately.

## API

### `BoundaryConfig.create(options)`

| Parameter | Type | Description |
|-----------|------|-------------|
| `server_keys` | `string[]` | Keys only accessible via `server()` |
| `client_keys` | `string[]` | Keys only accessible via `client()` |
| `load_options` | `LoadOptions` | Optional: standard DOTLYTE load options |

### `config.server()`

Returns a read-only config view restricted to server keys.

### `config.client()`

Returns a read-only config view restricted to client keys.

### `config.all_keys()`

Returns the full list of declared keys (both server and client).

## Framework Integration

### Next.js

```typescript
// lib/config.ts
import { createBoundaryConfig } from "dotlyte/boundary";

const boundary = createBoundaryConfig({
  serverKeys: ["DATABASE_URL", "STRIPE_SECRET", "JWT_SECRET"],
  clientKeys: ["NEXT_PUBLIC_API_URL", "NEXT_PUBLIC_APP_NAME"],
});

export const serverConfig = boundary.server();
export const clientConfig = boundary.client();
```

```typescript
// app/api/route.ts (server)
import { serverConfig } from "@/lib/config";
serverConfig.get("DATABASE_URL"); // ✅
```

```typescript
// components/Header.tsx (client)
import { clientConfig } from "@/lib/config";
clientConfig.get("NEXT_PUBLIC_APP_NAME"); // ✅
```

### Django

```python
# settings/config.py
from dotlyte import BoundaryConfig

boundary = BoundaryConfig.create(
    server_keys=["SECRET_KEY", "DATABASE_URL", "REDIS_URL"],
    client_keys=["SITE_NAME", "API_BASE_URL"],
)

server_config = boundary.server()
client_config = boundary.client()
```

## Error Messages

When a client accesses a server-only key:

```
Access denied: key 'DATABASE_URL' is not accessible from the client boundary.
Server-only keys must be accessed via config.server().
```
