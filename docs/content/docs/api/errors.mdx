DOTLYTE provides a hierarchy of typed errors with clear, actionable messages that include the key name, sources checked, and suggestions for resolution.

## Error Hierarchy

All DOTLYTE errors extend a base error type:

| Error Type | Cause | Extra Properties |
|---|---|---|
| `DotlyteError` | Base error | `key` |
| `MissingKeyError` | `require()` on missing key | `sources_checked` |
| `FileError` | File not found or unreadable | `file_path` |
| `ValidationError` | Schema validation failed | `violations` |
| `InterpolationError` | `${VAR}` resolution failed | — |
| `DecryptionError` | `ENC[...]` decryption failed | — |

<Callout type="info">
  Naming conventions vary by language: Python uses `MissingKeyError`, JavaScript uses `MissingKeyError`, Java/C# use `MissingKeyException`, etc. The structure is identical across all implementations.
</Callout>

### MissingKeyError

Thrown by `require()` / `require_keys()` when a key doesn't exist:

```python
config.require("database_url")
# MissingKeyError: Missing required config key 'database_url'.
# Checked sources: .env, config.yaml, environment variables.
# Suggestion: Add DATABASE_URL to your .env file or environment.
```

Extra properties:
- `sources_checked` / `sourcesChecked` / `SourcesChecked` — list of sources that were searched

---

### FileError

Thrown when a config file cannot be read (not found, permission denied, invalid syntax):

```python
config = load(files=["missing.env"])
# FileError: Config file not found: missing.env
```

Extra properties:
- `file_path` / `filePath` / `FilePath` — the file that caused the error

---

### ValidationError

Thrown by `assert_valid()` when schema validation fails:

```python
config.assert_valid({"port": {"type": "integer", "required": True}})
# ValidationError: Schema validation failed with 2 violations
```

Extra properties:
- `violations` / `Violations` — list of `SchemaViolation` objects, each with `key`, `message`, and `rule`

---

### InterpolationError

Thrown when `${VAR}` interpolation fails (circular reference, missing required variable):

```bash
# .env
A=${B}
B=${A}
```

```python
config = load()
# InterpolationError: Circular reference detected: A → B → A
```

---

### DecryptionError

Thrown when `ENC[...]` decryption fails (wrong key, corrupted data, no key available):

```python
config = load(encryption_key="wrong-key")
# DecryptionError: Failed to decrypt value for key 'api_secret'
```

## Handling Errors

### Python

```python
from dotlyte import load, DotlyteError, MissingKeyError, ValidationError

try:
    config = load(schema={"port": {"type": "integer", "required": True}})
    db_url = config.require("database_url")
except MissingKeyError as e:
    print(f"Missing key: {e.key}")
    print(f"Checked: {e.sources_checked}")
except ValidationError as e:
    for v in e.violations:
        print(f"  {v.key}: {v.message}")
except DotlyteError as e:
    print(f"Config error [{e.code}]: {e}")
```

### JavaScript

```typescript
import { load, MissingKeyError, ValidationError, DotlyteError } from "dotlyte";

try {
  const config = load({ schema: { port: { type: "integer", required: true } } });
  const dbUrl = config.require("database_url");
} catch (e) {
  if (e instanceof MissingKeyError) {
    console.error(`Missing: ${e.key}, checked: ${e.sourcesChecked}`);
  } else if (e instanceof ValidationError) {
    e.violations.forEach(v => console.error(`${v.key}: ${v.message}`));
  } else if (e instanceof DotlyteError) {
    console.error(`Config error [${e.code}]: ${e.message}`);
  }
}
```

### Go

```go
config, err := dotlyte.Load(nil)
if err != nil {
    var fileErr *dotlyte.FileError
    if errors.As(err, &fileErr) {
        log.Fatalf("File error: %s (path: %s)", fileErr.Message, fileErr.FilePath)
    }
    log.Fatal(err)
}

dbUrl, err := config.Require("database_url")
if err != nil {
    var missingErr *dotlyte.MissingKeyError
    if errors.As(err, &missingErr) {
        log.Fatalf("Missing key %s, checked: %v", missingErr.Key, missingErr.SourcesChecked)
    }
}
```

### Rust

```rust
use dotlyte::{load, DotlyteError};

match load(None) {
    Ok(config) => {
        match config.require("database_url") {
            Ok(url) => println!("DB: {}", url),
            Err(DotlyteError::MissingKey { key, sources_checked }) => {
                eprintln!("Missing: {key}, checked: {sources_checked:?}");
                std::process::exit(1);
            }
            Err(e) => panic!("{e}"),
        }
    }
    Err(DotlyteError::FileError { path, message }) => {
        eprintln!("File error: {path}: {message}");
    }
    Err(e) => eprintln!("Config error: {e}"),
}
```

### Java

```java
import dev.dotlyte.*;

try {
    Config config = Dotlyte.load(LoadOptions.builder().build());
    Object dbUrl = config.require("database_url");
} catch (DotlyteException.MissingKeyException e) {
    System.err.println("Missing: " + e.getKey());
    System.err.println("Checked: " + e.getSourcesChecked());
} catch (DotlyteException.ValidationException e) {
    e.getViolations().forEach(v ->
        System.err.println(v.getKey() + ": " + v.getMessage()));
} catch (DotlyteException e) {
    System.err.println("Config error: " + e.getMessage());
}
```

### C\#

```csharp
using Dotlyte;

try
{
    var config = DotlyteLoader.Load();
    var dbUrl = config.Require("database_url");
}
catch (MissingKeyException ex)
{
    Console.Error.WriteLine($"Missing: {ex.Key}");
    Console.Error.WriteLine($"Checked: {string.Join(", ", ex.SourcesChecked)}");
}
catch (ValidationException ex)
{
    foreach (var v in ex.Violations)
        Console.Error.WriteLine($"{v.Key}: {v.Message}");
}
catch (DotlyteException ex)
{
    Console.Error.WriteLine($"Config error: {ex.Message}");
}
```

## Best Practices

<Callout type="info">
  **Fail fast at startup.** Call `require()` for all critical config at application startup, not lazily at runtime. This ensures missing config is caught immediately.
</Callout>

```python
# Good: Validate all required config at startup
config = load(schema={
    "database_url": {"type": "string", "required": True},
    "jwt_secret": {"type": "string", "required": True, "sensitive": True},
    "port": {"type": "integer", "default": 8080},
})
config.assert_valid()

DB_URL = config.require("database_url")
SECRET = config.require("jwt_secret")
PORT = config.get("port", 8080)

# Bad: Lazy access deep in request handler
def handle_request():
    db = connect(config.require("database_url"))  # Might fail on first request!
```
