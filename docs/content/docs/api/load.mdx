`load()` is the primary function in DOTLYTE. It discovers, parses, merges, coerces, interpolates, decrypts, and validates configuration from all available sources and returns a `Config` object.

## Parameters

See [LoadOptions](/docs/api/options) for the full options reference.

## Return Value

Returns a [`Config`](/docs/api/config) object with dot-notation access, `get()`, `require()`, `has()`, schema validation, serialization, and more.

## Basic Usage

```python
from dotlyte import load

# Load with all defaults
config = load()

# Load with options
config = load(
    env="production",
    prefix="APP",
    defaults={"timeout": 30},
    schema={"port": {"type": "integer", "required": True}},
)
```

## Behavior

When called, `load()` performs these steps in order:

1. **Discover** config files in the current working directory (or walk up with `find_up`)
2. **Parse** each file using the appropriate parser (dotenv, YAML, JSON, TOML)
3. **Parse** environment variables (optionally filtered by `prefix`)
4. **Merge** all sources by [priority order](/docs/priority-order), with `overrides` on top
5. **Interpolate** `${VAR}` references (unless `interpolate_vars=False`)
6. **Apply schema defaults** from `schema` rules
7. **Coerce** string values to proper types ([type coercion rules](/docs/type-coercion))
8. **Decrypt** `ENC[...]` values if an encryption key is available
9. **Validate** against `schema` rules (throw on failure if `strict`)
10. **Detect** sensitive keys for masking
11. **Start watcher** if `watch=True` (Python, JS, Go, Rust, Java, Ruby only)
12. **Return** a `Config` object

## Examples

### Minimal

```python
config = load()
```

### With Environment

```python
config = load(env="staging")
# Also loads: config.staging.yaml, .env.staging, etc.
```

### With Prefix Stripping

```python
config = load(prefix="MYAPP")
# MYAPP_DB_HOST → db_host
```

### With Schema Validation

```python
config = load(
    schema={
        "port": {"type": "integer", "required": True, "min": 1, "max": 65535},
        "host": {"type": "string", "required": True},
        "api_key": {"type": "string", "sensitive": True},
    },
    strict=True,  # reject unknown keys
)
```

### With Variable Interpolation

```bash
# .env
BASE_URL=https://api.example.com
API_ENDPOINT=${BASE_URL}/v1
GREETING=${NAME:-World}
```

```python
config = load()
config.get("api_endpoint")  # "https://api.example.com/v1"
config.get("greeting")      # "World"
```

### With Encryption

```python
config = load(encryption_key="your-64-char-hex-key")
# ENC[...] values in .env files are auto-decrypted
```

### With File Watching (Python, JS, Go, Rust, Java, Ruby)

```python
config = load(watch=True, debounce_ms=1000)
config.on_change(lambda old, new: print("Config changed!"))
# Call config.close() when done
```

### With Overrides

```python
config = load(
    defaults={"port": 3000},
    overrides={"port": 9999},  # highest priority — always wins
)
config.get("port")  # 9999
```

### With Find-Up Discovery

```python
config = load(find_up=True, root_markers=[".git", "pyproject.toml"])
# Walks up from cwd until a root marker is found
```

### With Custom Sources

```python
config = load(sources=["defaults", "dotenv", "env"])
# Only loads defaults, .env files, and env vars
# Skips YAML, JSON, TOML
```

### With Explicit Files

```python
config = load(files=["config/app.env", "config/secrets.env"])
```
