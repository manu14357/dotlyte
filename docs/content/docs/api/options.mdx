All options are **optional**. Calling `load()` with no arguments uses sensible defaults.

## Core Options

### `files`

**Type:** `string[]` | **Default:** `null`

Explicit list of files to load. When specified, automatic file discovery is skipped.

```python
config = load(files=["config/app.env", "config/database.env"])
```

### `prefix`

**Type:** `string` | **Default:** `null`

Strip this prefix from environment variable names. The prefix and separator (`_`) are removed, and the remaining key is lowercased.

```python
# Given: MYAPP_DB_HOST=localhost
config = load(prefix="MYAPP")
config.get("db_host")  # "localhost"
```

### `defaults`

**Type:** `dict` / `Record<string, unknown>` / `map[string]any` | **Default:** `{}`

Default values used as the lowest-priority layer. Useful for providing fallback configuration.

```python
config = load(defaults={
    "port": 3000,
    "host": "0.0.0.0",
    "database": {
        "host": "localhost",
        "port": 5432,
    }
})
```

### `sources`

**Type:** `string[]` | **Default:** `null`

Custom source loading order. When specified, only the listed sources are loaded in the given order.

Valid values: `"defaults"`, `"toml"`, `"yaml"`, `"json"`, `"dotenv"`, `"env"`

```python
# Only load defaults and env vars (skip all files)
config = load(sources=["defaults", "env"])
```

### `env`

**Type:** `string` | **Default:** `null`

Environment name. When specified, environment-specific files are also loaded:

- `config.{env}.yaml` / `config.{env}.yml`
- `config.{env}.json`
- `config.{env}.toml`
- `.env.{env}`

```python
config = load(env="production")
# Also loads: config.production.yaml, .env.production, etc.
```

### `overrides`

**Type:** `dict` / `Record<string, unknown>` | **Default:** `{}`

Values applied with the **highest priority** — they override everything, including environment variables. Useful for CLI flags or programmatic overrides.

```python
config = load(
    defaults={"port": 3000},
    overrides={"port": 9999},
)
config.get("port")  # 9999 (overrides always win)
```

## Schema & Validation Options

### `schema`

**Type:** `dict<string, SchemaRule>` | **Default:** `null`

Define validation rules, default values, type constraints, and sensitivity markers per key.

```python
config = load(schema={
    "port": {"type": "integer", "required": True, "min": 1, "max": 65535},
    "host": {"type": "string", "required": True, "format": "ipv4"},
    "api_key": {"type": "string", "required": True, "sensitive": True},
    "log_level": {"type": "string", "enum": ["debug", "info", "warn", "error"], "default": "info"},
})
```

Schema rule fields: `type`, `required`, `format`, `pattern`, `enum`/`enum_values`, `min`, `max`, `default`/`default_value`, `sensitive`, `doc`.

### `strict`

**Type:** `bool` | **Default:** `false`

When `true`, validation rejects keys in the config that are **not defined** in the schema. Useful for catching typos.

```python
config = load(
    schema={"port": {"type": "integer"}},
    strict=True,
)
# Throws ValidationError if config contains keys not in the schema
```

## Interpolation & Encryption Options

### `interpolate_vars` / `interpolateVars`

**Type:** `bool` | **Default:** `true`

Enable `${VAR}` variable interpolation in config values.

```bash
# .env
BASE_URL=https://api.example.com
API_ENDPOINT=${BASE_URL}/v1
GREETING=${NAME:-default_value}
REQUIRED=${SECRET:?error message}
LITERAL=$$NOT_INTERPOLATED
```

Set to `false` to treat `${...}` as literal text.

### `encryption_key` / `encryptionKey`

**Type:** `string` | **Default:** `null`

A 64-character hex string (AES-256 key) for decrypting `ENC[...]` values. If not provided, the key is resolved from: `DOTLYTE_KEY_{ENV}` → `DOTLYTE_KEY` → `.dotlyte-keys` file.

```python
config = load(encryption_key="aabbccddee...")
```

## File Discovery Options

### `find_up` / `findUp`

**Type:** `bool` | **Default:** `false`

Walk up parent directories searching for config files, stopping when a root marker is found.

### `root_markers` / `rootMarkers`

**Type:** `string[]` | **Default:** `[".git", "package.json", "go.mod", "Cargo.toml", ".dotlyte"]`

Files or directories that mark the project root when using `find_up`.

### `cwd`

**Type:** `string` | **Default:** `null` (uses process working directory)

Override the working directory for file discovery.

```python
config = load(cwd="/app/config", find_up=True)
```

## Advanced Options

### `allow_all_env_vars` / `allowAllEnvVars`

**Type:** `bool` | **Default:** `false`

By default, DOTLYTE filters environment variables to avoid loading system variables. Set to `true` to include all env vars.

### `debug`

**Type:** `bool` | **Default:** `false`

Enable debug logging to see which files are discovered, which sources are loaded, and how values are merged.

### `watch`

**Type:** `bool` | **Default:** `false`

**Supported in:** Python, JavaScript, Go, Rust, Java, Ruby (not PHP, not .NET)

Start a file watcher that reloads config when source files change. Use `config.on_change()` to register callbacks.

```python
config = load(watch=True)
config.on_change(lambda old, new: print("Config reloaded"))
config.close()  # Stop watching when done
```

### `debounce_ms` / `debounceMs`

**Type:** `int` | **Default:** `500`

Milliseconds to debounce file change events when `watch` is enabled.
