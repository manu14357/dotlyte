All options are **optional**. Calling `load()` with no arguments uses sensible defaults.

## Options

### `files`

**Type:** `string[]` | **Default:** `undefined`

Explicit list of files to load. When specified, automatic file discovery is skipped.

```python
config = load(files=["config/app.env", "config/database.env"])
```

### `prefix`

**Type:** `string` | **Default:** `undefined`

Strip this prefix from environment variable names. The prefix and separator (`_`) are removed, and the remaining key is lowercased.

```python
# Given: MYAPP_DB_HOST=localhost
config = load(prefix="MYAPP")
config.db_host  # "localhost"
```

### `defaults`

**Type:** `dict` / `Record<string, unknown>` / `map[string]any` | **Default:** `{}`

Default values used as the lowest-priority layer. Useful for providing fallback configuration.

```python
config = load(defaults={
    "port": 3000,
    "host": "0.0.0.0",
    "database": {
        "host": "localhost",
        "port": 5432,
    }
})
```

### `sources`

**Type:** `string[]` | **Default:** `undefined`

Custom source loading order. When specified, only the listed sources are loaded in the given order.

Valid values: `"defaults"`, `"toml"`, `"yaml"`, `"json"`, `"dotenv"`, `"env"`

```python
# Only load defaults and env vars (skip all files)
config = load(sources=["defaults", "env"])
```

### `env`

**Type:** `string` | **Default:** `undefined`

Environment name. When specified, environment-specific files are also loaded:

- `config.{env}.yaml` / `config.{env}.yml`
- `config.{env}.json`
- `config.{env}.toml`
- `.env.{env}`

```python
config = load(env="production")
# Also loads: config.production.yaml, .env.production, etc.
```
